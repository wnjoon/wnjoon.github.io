## Oracle Database

오라클 데이터베이스(Oracle Database 또는 Oracle RDBMS)는 미국 오라클(Oracle)사의 관계형 데이터베이스 관리 시스템의 이름이다. 현재 유닉스 환경에서 가장 널리 사용되는 RDBMS이다. 검색이나 업데이트용 언어로는 국제표준화기구의 표준 구조화 조회 언어와 PL/SQL을 지원한다. [위키백과](https://ko.wikipedia.org/wiki/%EC%98%A4%EB%9D%BC%ED%81%B4_%EB%8D%B0%EC%9D%B4%ED%84%B0%EB%B2%A0%EC%9D%B4%EC%8A%A4)  

<br>


## Oracle 11g 도커 구성하기

### 1. 도커 이미지 다운로드

**도커 이미지를 다운로드하기 위해 도커 홈페이지에 로그인해야 할 수도 있다.**

```sh
$ docker search oracle-xe-11g
$ docker pull jaspeen/oracle-xe-11g
```

### 2. 이미지 실행

맥에서는 OS 별(M1, x86/64)로 도커 이미지를 구동하는 방법에 차이가 있다. x86/64의 경우에는 일반적인 도커 이미지 구동방식을 그대로 사용하여도 무방하나, M1의 경우에는 Colima를 활용하여 구동하는 것을 권장한다. 그렇지 않으면 Oracle Cloud Database 또는 AWS와 같은 클라우드에 직접 컨테이너를 띄워서 원격접속해야 한다.  

Colima 사용 방법은 필자의 블로그 ['M1에서 x86/64 기반의 컨테이너 구동하기'](../docker/2022-09-04-m1oracle.md)를 참조한다. 더 자세한 내용을 확인하고자 한다면, 필자의 블로그 글을 작성할 수 있도록 도와준 ['[MacOS] M1 맥북 도커로 ORACLE DB 실행하기 - Shane's Planet'](https://shanepark.tistory.com/400) 글을 정독하길 바란다.

Colima 구동이 완료된 이후 도커 이미지 실행은 다음과 같다.

```sh
$ docker run --name oracle11g -d -p 1521:1521 jaspeen/oracle-xe-11g
```

### 3. 컨테이너 접속

```sh
$ docker exec -it oracle11g sqlplus
```

기본 접속 정보는 아래와 같다.
- user name : system
- password : oracle

> ORA-01089: immediate shutdown in progress - no operations are permitted   
> => 컨테이너가 아직 제대로 만들어지지 않은 상태이므로, 약간의 시간이 지나고 다시 접속해본다.

## DBeaver 연결

필자는 [DBeaver](https://dbeaver.io/)를 이용하여 데이터베이스를 연결하였다.  
DBeaver에서 접속 환경에 필요한 내용은 아래와 같다.  
- Host : 오라클 데이터베이스 컨테이너의 위치. 로컬호스트 또는 해당 클라우드의 퍼블릭 IPv4가 될 것이다.
- Port : 1521 (위에서 이와 같이 생성했으므로)
- Database : 오라클 데이터베이스에 접속한 후 아래의 명령어를 통해 확인할 수 있다.

## ORM을 지원하지 않는 오라클

중요한 점이, 오라클은 ORM(Object Relational Mapping)을 지원하지 않는다.  

ORM이란, 객체와 관계형 데이터베이스의 데이터를 자동으로 연결해주는 방식이다. 즉 SQL을 하나씩 작성하지 않고 일반 프로그래밍 언어에서의 구조체(struct) 또는 클래스(class)를 그대로 사용하여 데이터베이스와 통신할 수 있게 된다.  

참고로 Golang은 클래스가 없기 때문에 구조체가 이 역할을 대신하게 되지만, 어찌되었든 오라클 자체가 ORM을 **공식적으로** 지원하지 않으므로 있는 그대로의 SQL문을 작성해야만 한다.

## 사용 방법

### 1. 사용자 생성 및 권한 부여

오라클에서 사용할 사용자를 생성한다. 오라클 컨테이너에서 sqlplus로 들어가서 실행한다.

```sh
# ID : oracleuser
# PW : oracleuserpw
SQL> create user oracleuser identified by oracleuserpw;

User created.
```

다음으로는 해당 계정에 권한을 부여한다.

```sh
SQL> grant connect, resource, dba to oracleuser;

Grant succeeded.
```

<u>참고로 위의 과정들도 모두 Golang의 함수로 작성할 수 있다.</u>

### 2. 연결

```go
import (
	"database/sql"			
)

func main {
  // oracle://{계정아이디}:{계정비밀번호}@{호스트IP}:{Port}/{서비스이름}
  // 서비스이름은 위의 DBeaver 연결 과정에서 도출한 Database 항목과 일치한다.
  connection := "oracle://oracleuser:oracleuserpw@34.123.45.678:1521/XE"            
  db, err := sql.Open("oracle", connection())
}
```

### 3. SQL

#### 3.1. 기본 실행

```go
func main {
  // connect 이후 진행  
  db.Exec("CREATE TABLE USER_TABLE ( NAME VARCHAR2(100), COMPANY VARCHAR2(1000), ADDRESS VARCHAR2(100))")
}
```

위와 같이 오라클에서 사용하는 SQL문을 db.Exec()내부에 넣어주기만 하면 된다.  
이러한 값들을 외부에서 파라미터로 받아서 쓰고싶다면, <u>fmt.Sprintf</u>를 사용하면 된다.  

```go
func CreateTable(name string) {
  // connect 이후
  createTable := "CREATE TABLE %s ( NAME VARCHAR2(100), COMPANY VARCHAR2(1000), ADDRESS VARCHAR2(100))"
  sql := fmt.Sprintf(createTable, name)
  db.Exec(sql)
}
```

#### 3.2. INSERT

파라미터로 전달된 값을 테이블에 INSERT하는 과정은 추가적인 작업이 요구된다. 하지만 매우 간단하다.  

```go
func AddUser(name, company, address, table string) {
  // connect 이후
  // table 파라미터에 입력할 테이블을 입력
  // Error handling 제외
  insertRow := "INSERT INTO %s ( NAME, COMPANY, ADDRESS) VALUES (:name, :company, :address)"
  sql := fmt.Sprintf(insertRow, table)
  stmt, err := db.Prepare(sql)
  execResult, err := stmt.Exec(name, company, address)

  // INSERT 문을 통해 몇개의 열에 변경이 일어났는지 알려주는 부분
  row, err := execResult.RowsAffected()
}
```

#### 3.3. SELECT

쿼리하는 부분은 또 조금 다르다. 먼저 받을 값에 대한 변수를 만들어놓고, 그 곳에 반환된 값을 저장하는 방식이다.

```go
func getUser(name, tableName string) (string, string){
  // connect 이후
  // Error handling 제외
  
  // 받을 값에 대한 변수 미리 선언(필수)
  var (
		company string
		address string
	)

  selectRow := "SELECT COMPANY, ADDRESS FROM %s WHERE NAME = '%s'"
  sql := fmt.Sprintf(selectRow, tableName, name)
  row := db.QueryRow(sql)
  err := row.Scan(&company, &address)
  
  return company, address
}
```
