2021л…„ 8мӣ” 4мқј вҖў tool вҖў k8s, к°•мқҳ


мһҗмӣҗ л¶„л°°мқҳ н•„мҡ”м„ұ

мҝ лІ„л„ӨнӢ°мҠӨлҠ” нҒҙлҹ¬мҠӨн„°м—җ л”°лқј мӮ¬мҡ©н•  мҲҳ мһҲлҠ” мһҗмӣҗмқҳ н•ңкі„к°Җ мһҲлӢӨ. л§Ҳм№ҳ мҡ°лҰ¬к°Җ нҒҙлқјмҡ°л“ң нҳ№мқҖ мҷёмһҘн•ҳл“ңмҷҖ к°ҷмқҖ лі„лҸ„мқҳ мһҘм№ҳлӮҳ м„ңл№„мҠӨлҘј мӮ¬мҡ©н•ҳм§Җ м•ҠлҠ”лӢӨл©ҙ PCм—җ м—°кІ°лҗң н•ҳл“ң лҳҗлҠ” SSDмқҳ мҡ©лҹүл§Ң мӮ¬мҡ©н•  мҲҳ мһҲлҠ” кІғкіј к°ҷлӢӨ. м—¬кё°м„ңмқҳ мһҗмӣҗмқҖ CPU, MemoryмҷҖ к°ҷмқҖ мў…лҘҳлҘј нҸ¬н•Ён•ңлӢӨ.


@к·ёлҰј 1: н•ңм •лҗң мһҗмӣҗмқ„ л„Өмһ„мҠӨнҺҳмқҙмҠӨмҷҖ нҢҢл“ңм—җ м Ғм ҲнһҲ л¶„л°°н•ҙм•јл§Ң н•ңлӢӨ. - м¶ңмІҳ: к°•мқҳмһҗлЈҢ

мҝ лІ„л„ӨнӢ°мҠӨ нҒҙлҹ¬мҠӨн„°лҠ” лӢӨм–‘н•ң л„Өмһ„мҠӨнҺҳмқҙмҠӨмҷҖ мқҙлҹ¬н•ң л„Өмһ„мҠӨнҺҳмқҙмҠӨ лӮҙл¶Җм—җ мЎҙмһ¬н•ҳлҠ” м—¬лҹ¬ к°ңмқҳ нҢҢл“ңл“Өмқ„ кө¬м„ұн•  мҲҳ мһҲлӢӨ. мқҙл“ӨмқҖ нҒҙлҹ¬мҠӨн„°мқҳ мһҗмӣҗмқ„ 'лӮҳлҲ„м–ҙм„ң' мӮ¬мҡ©н•ҳкІҢ лҗҳлҠ”лҚ°, мқҙлҹ¬н•ң мқҙмң лЎң мһҗмӣҗмқ„ нҡЁмңЁм ҒмңјлЎң л¶„л°°н•ҙм„ң лҗҳлҸ„лЎқ л§ҺмқҖ нҢҢл“ңк°Җ м„ңл№„мҠӨлҘј мҲҳн–үн•  мҲҳ мһҲлҸ„лЎқ н•ҙм•ј н•ңлӢӨ. л§Ңм•Ҫ н•ҳлӮҳмқҳ л„Өмһ„мҠӨнҺҳмқҙмҠӨм—җм„ң лӘЁл“  мһҗмӣҗмқ„ мӮ¬мҡ©н•ҙлІ„лҰ°лӢӨл©ҙ, нҒҙлҹ¬мҠӨн„° лӮҙмқҳ лӢӨлҘё л„Өмһ„мҠӨнҺҳмқҙмҠӨм—җ мҶҚн•ң нҢҢл“ңл“ӨмқҖ мһҗмӣҗмқ„ н•ҳлӮҳлҸ„ мӮ¬мҡ©н•ҳм§Җ лӘ»н•ҳкі  м„ңл№„мҠӨк°Җ мқҙлЈЁм–ҙм§Җм§Җ лӘ»н•  кІғмқҙлӢӨ.

мқҙлҘј л°©м§Җн•ҳкё° мң„н•ҙ ResourceQuotaмҷҖ LimitRangeлқјлҠ” мҳөм…ҳмқҙ мһҲлӢӨ. ResourceQuotaлҠ” л„Өмһ„мҠӨнҺҳмқҙмҠӨм—җ мөңлҢҖлЎң н• лӢ№ к°ҖлҠҘн•ң мһҗмӣҗмқҳ нҒ¬кё°лҘј н•ңм •н•ҳлҠ” кё°лҠҘмқҙкі , LimitRangeлҠ” л„Өмһ„мҠӨнҺҳмқҙмҠӨм—җ мһҲлҠ” мһҗмӣҗмқ„ нҠ№м • нҢҢл“ңк°Җ лӘЁл‘җ мӮ¬мҡ©н•ҳм§Җ м•ҠлҸ„лЎқ л„Өмһ„мҠӨнҺҳмқҙмҠӨм—җ н• лӢ№н•  мҲҳ мһҲлҠ” нҢҢл“ңмқҳ мһҗмӣҗ нҒ¬кё°лҘј м ңн•ңн•ңлӢӨ. л¬јлЎ  ResourceQuotaмҷҖ LimitRangeлҠ” л„Өмһ„мҠӨнҺҳмқҙмҠӨм—җл§Ң м„Өм •н•  мҲҳ мһҲлҠ” кІғмқҙ м•„лӢҲл©°, нҒҙлҹ¬мҠӨн„° м „мІҙм—җ лҢҖн•ҙм„ңлҸ„ м„Өм •н•  мҲҳ мһҲлӢӨ.


Namespace

NamespaceлҠ” мҝ лІ„л„ӨнӢ°мҠӨ нҒҙлҹ¬мҠӨн„° лӮҙл¶Җм—җ мЎҙмһ¬н•ҳлҠ” лҳҗ лӢӨлҘё к°ҖмғҒ нҒҙлҹ¬мҠӨн„° кіөк°„мқҙлӢӨ. м„ңлЎң лӢӨлҘё Namespaceмқҳ нҢҢл“ңлҒјлҰ¬лҠ” Network PolicyлҘј мқҙмҡ©н•ҙм„ңл§Ң нҶөмӢ мқҙ к°ҖлҠҘн•ҳлӢӨ. Namespace м•Ҳм—җ мғқм„ұлҗң лӘЁл“  мҳӨлёҢм қнҠё(нҢҢл“ң, м„ңл№„мҠӨ л“ұ)лҠ” Namespaceм—җ мў…мҶҚм Ғмқҙкё° л•Ңл¬ём—җ, Namespaceк°Җ мӮӯм ңлҗҳл©ҙ н•ҙлӢ№ мҳӨлёҢм қнҠё лҳҗн•ң мӮӯм ңлҗңлӢӨ.


@к·ёлҰј 2: Namespaceм—җ лҢҖн•ң к°„лһөн•ң мҳҲмӢң - м¶ңмІҳ: к°•мқҳмһҗлЈҢ

1. Namespace мғқм„ұ

к·ёлҰј 2м—җм„ңмқҳ 1-1, 1-1вҖҷлҘј лӮҳнғҖлӮёлӢӨ. мҳҲмӢңм—җм„ңлҠ” 1-1мқҳ nm-1 NamespaceлҘј мғқм„ұн•ңлӢӨ.

# 1-1
apiVersion: v1
kind: Namespace
metadata:
  name: nm-1

cliлҘј мқҙмҡ©н•ҳм—¬ мғқм„ұн•  мҲҳлҸ„ мһҲлӢӨ.

kubectl create namespace nm-1

2. Namespace лӮҙ мҳӨлёҢм қнҠё(нҢҢл“ң, м„ңл№„мҠӨ) мғқм„ұ

к·ёлҰј 2м—җм„ңмқҳ 1-2, 1-3, 1-2вҖҷлҘј лӮҳнғҖлӮёлӢӨ.

  • мҳӨлёҢм қнҠёлҘј мғқм„ұн•  л•Ң NamespaceлҘј лӘ…мӢңн•ҙмЈјм§Җ м•Ҡмңјл©ҙ, кё°ліёк°’мқё default Namespaceм—җ мҳӨлёҢм қнҠёк°Җ мғқм„ұлҗңлӢӨ.
  • 1-2м—җм„ң ліј мҲҳ мһҲлҠ” кІғ мІҳлҹј, Namespace м•Ҳм—җлҠ” лҸҷмқјн•ң мқҙлҰ„мқҳ нҢҢл“ңк°Җ мЎҙмһ¬н•  мҲҳ м—ҶлӢӨ.
  • м„ңлЎң лӢӨлҘё Namespaceм—җ мЎҙмһ¬н•ҳлҠ” нҢҢл“ңлҒјлҰ¬ м§Ғм ‘ нҶөмӢ н•ҳл Өл©ҙ Network policyлҘј лі„лҸ„лЎң мӮ¬мҡ©н•ҙм•ј н•ңлӢӨ.
  • м„ңл№„мҠӨ лҳҗн•ң кё°ліём ҒмңјлЎңлҠ” л§Ңл“Өм–ҙм§„ Namespace м•Ҳм—җм„ңл§Ң лҸҷмһ‘н•  мҲҳ мһҲм§Җл§Ң, PV лҳҗлҠ” NodePortмҷҖ к°ҷмқҙ лӘЁл“  Namespaceм—җм„ң мӮ¬мҡ© к°ҖлҠҘн•ң мҳӨлёҢм қнҠёлҸ„ мЎҙмһ¬н•ңлӢӨ.
  • NamespaceлҘј м§Җмҡ°л©ҙ, лӮҙл¶Җм—җ мһҲлҠ” мҳӨлёҢм қнҠё лҳҗн•ң лӘЁл‘җ мӮӯм ңлҗңлӢӨ.

    app: pod

ResourceQuota


@к·ёлҰј 3: ResourceQuota лҢҖн•ң к°„лһөн•ң мҳҲмӢң - м¶ңмІҳ: к°•мқҳмһҗлЈҢ

ResourceQuotaлҠ” мң„м—җм„ң м„ӨлӘ…н•ң кІғкіј к°ҷмқҙ, Namespaceм—җм„ң мӮ¬мҡ© к°ҖлҠҘн•ң мһҗмӣҗ(CPU, Memory, Storage л“ұ)мқҳ н•ңкі„лҘј м •н•ңлӢӨ. ResourceQuotaк°Җ л§Ңл“Өм–ҙм§„ мқҙнӣ„м—җ мғқм„ұлҗҳлҠ” лӘЁл“  нҢҢл“ңлҠ” ResourceQuotaк°Җ мһ‘м„ұлҗҳм–ҙ мһҲм–ҙм•ј н•ҳл©°, мқҙм—җ лҢҖн•ң кё°мӨҖмқҙ л§һм•ҳмқ„ кІҪмҡ°м—җл§Ң нҢҢл“ңк°Җ мғқм„ұлҗңлӢӨ.

1. ResourceQuota мғқм„ұ

к·ёлҰј 3м—җм„ңмқҳ 2-2лҘј лӮҳнғҖлӮёлӢӨ.

  • requests: м»Ён…Ңмқҙл„ҲлҘј мғқм„ұн•  л•Ң мҡ”мІӯ к°ҖлҠҘн•ң мөңлҢҖ мһҗмӣҗ нҒ¬кё°
  • limits: мғқм„ұлҗң м»Ён…Ңмқҙл„Ҳк°Җ м¶”к°ҖлЎң мӮ¬мҡ©н•  мҲҳ мһҲлҠ” мһҗмӣҗмқҳ н•ңкі„м№ҳ
# 2-2
apiVersion: v1
kind: ResourceQuota
metadata:
  name: rq
  namespace: nm-2
spec:
  hard: # н•ңкі„м№ҳлҘј лӘ…мӢңн•ҳлҠ” л¶Җл¶„
    requests.memory: 1Gi
    limits.memory: 1Gi

ResourceQuotaлҠ” мһҗмӣҗмқҳ н•ңкі„м№ҳ лҝҗл§Ң м•„лӢҲлқј, Namespaceм—җ мғқм„ұк°ҖлҠҘн•ң мҳӨлёҢм қнҠёмқҳ мҲҳлҸ„ м ңн•ңн•  мҲҳ мһҲлӢӨ. RequestQuotaк°Җ мһҳ мғқм„ұлҗҳм—ҲлҠ”м§Җ нҷ•мқён•ҳл Өл©ҙ м•„лһҳмқҳ лӘ…л №м–ҙлҘј мӮ¬мҡ©н•ңлӢӨ.

kubectl describe resourcequotas --namespace=nm-2

2. нҢҢл“ң мғқм„ұ

# 2-4
apiVersion: v1
kind: Pod
metadata:
  name: pod
spec:
  containers:
  - name: container
    image: nginx
    resources:  # мқҙ л¶Җл¶„мқҙ лӘ…мӢңлҗҳм§Җ м•Ҡмңјл©ҙ нҢҢл“ңк°Җ мғқм„ұлҗҳм§Җ м•ҠлҠ”лӢӨ.
      requests:
        memory: 0.5Gi 
      limits:
        memory: 0.5Gi

л§Ңм•Ҫ namespace лӮҙ лӘЁл“  нҢҢл“ңмқҳ requestsмҷҖ limitsмқҳ н•©мқҙ requestQuotaм—җ м„Өм •н•ң к°’ліҙлӢӨ мҙҲкіјн•ңлӢӨл©ҙ, нҢҢл“ңлҠ” мғқм„ұлҗҳм§Җ м•ҠлҠ”лӢӨ.


LimitRange


@к·ёлҰј 4: LimitRange лҢҖн•ң к°„лһөн•ң мҳҲмӢң - м¶ңмІҳ: к°•мқҳмһҗлЈҢ

к°Ғ нҢҢл“ңк°Җ Namespaceм—җ л“Өм–ҙмҳ¬ мҲҳ мһҲлҠ”м§Җ м—¬л¶ҖлҘј нҷ•мқён•  л•Ң мӮ¬мҡ©н•ңлӢӨ. к·ёлҰј 4мқҳ 3-2к°Җ мқҙм—җ н•ҙлӢ№н•ңлӢӨ.

  • min: нҢҢл“ңк°Җ м¶©мЎұн•ҙм•ј н•ҳлҠ” мһҗмӣҗмқҳ мөңмҶҢ нҒ¬кё°
  • max: нҢҢл“ңк°Җ м¶©мЎұн•ҙм•ј н•ҳлҠ” мһҗмӣҗмқҳ мөңлҢҖ нҒ¬кё°. к·ёлҰјм—җм„ң нҢҢл“ң(3-3)мқҖ limitмқҳ к°’мқҙ maxлҘј мҙҲкіјн•ҳкё° л•Ңл¬ём—җ мғқм„ұлҗҳм§Җ м•ҠлҠ”лӢӨ.
  • maxLimitRequestRatio: requestмҷҖ limitмқҳ мөңлҢҖ л№„мңЁ. к·ёлҰјм—җм„ң нҢҢл“ң(3-3)мқҖ requestмҷҖ limitмқҳ л№„мңЁмқҙ 5мқҙлҜҖлЎң, 3ліҙлӢӨ нҒ¬кё° л•Ңл¬ём—җ мғқм„ұлҗҳм§Җ м•ҠлҠ”лӢӨ.
  • defaultRequest: requestмҷҖ limitлҘј лӘ…мӢңн•ҳм§Җ м•Ҡкі  нҢҢл“ңлҘј мғқм„ұн•  л•Ң кё°ліёк°’. к·ёлҰјмқҳ 3-3вҖҷмқҙ мқҙм—җ н•ҙлӢ№н•ңлӢӨ.
  • Namespace л°Ҹ нҒҙлҹ¬мҠӨн„°м—җ м Ғмҡ© к°ҖлҠҘн•ң ResourceQuotaмҷҖ лӢӨлҘҙкІҢ LimitRangeлҠ” мҳӨм§Ғ namespace лӮҙм—җм„ңл§Ң мӮ¬мҡ© к°ҖлҠҘн•ҳлӢӨ.

1. LimitRange мғқм„ұ

# 3-2
apiVersion: v1
kind: LimitRange
metadata:
  name: lr
spec:
  limits:
  - type: Container
    min:
      memory: 0.1Gi
    max:
      memory: 0.4Gi
    maxLimitRequestRatio:
      memory: 3
    defaultRequest:
      memory: 0.1Gi
    default:
      memory: 0.2Gi

RequestQuotaк°Җ мһҳ мғқм„ұлҗҳм—ҲлҠ”м§Җ нҷ•мқён•ҳл Өл©ҙ м•„лһҳмқҳ лӘ…л №м–ҙлҘј мӮ¬мҡ©н•ңлӢӨ.

kubectl describe limitranges --namespace={namespace мқҙлҰ„}

3.LimitRangeк°Җ мӨ‘ліөлҗңлӢӨл©ҙ?

л§Ңм•Ҫ н•ҳлӮҳмқҳ Namespace лӮҙм—җ м„ңлЎң лӢӨлҘё requestмҷҖ limitлҘј к°–лҠ” LimitRangeк°Җ 2к°ң мқҙмғҒ мЎҙмһ¬н•ңлӢӨл©ҙ м–ҙл–»кІҢ лҗ к№Ң? мҳҲмӢңлЎң maxлҠ” лҚ” мһ‘мқҖ к°’мқ„ кё°мӨҖмқҙлқјкі  нҢҗлӢЁн•ҳлҠ”лҚ°, мқҙ кІҪмҡ° нҢҢл“ңлҘј лҚ” нҒ° maxлЎң л§һм¶”кі  мғқм„ұн•ҳл©ҙ мғқм„ұмқҙ м•Ҳлҗ  мҲҳ мһҲлӢӨ. мқҙлҹ¬н•ң нҳјм„ мқ„ л°©м§Җн•ҳкё° мң„н•ҙ LimitRangeлҠ” кјӯ м•ҪмҶҚлҗң н•ҳлӮҳмқҳ кё°мӨҖмңјлЎң м Ғмҡ©лҗ  мҲҳ мһҲлҸ„лЎқ н•ҙм•ј н•ңлӢӨ.
